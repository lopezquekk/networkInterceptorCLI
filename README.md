# ni — Network Interceptor

A lightweight, Charles-like HTTP/HTTPS traffic interceptor for macOS, built on [mitmproxy](https://mitmproxy.org/). Captures system-wide network traffic to a local SQLite database and lets you mock any endpoint with a JSON file — all from a single CLI.

## Features

- **System-wide interception** — sets macOS HTTP/HTTPS proxy automatically for every active network service
- **HTTPS decryption** — installs a trusted CA certificate into the System keychain so browsers (Chrome, Arc, Safari) decrypt TLS without warnings
- **Capture store** — every request/response is saved to `~/.networkinterceptor/captures.db` with full headers and bodies
- **Glob-pattern mocking** — intercept any URL pattern and return a local JSON file instead of the real response
- **iOS Simulator support** — installs the CA cert into booted simulators with a single command; simulators inherit the system proxy automatically
- **CLI-first** — single `./ni` entry point with intuitive subcommands; no GUI required

## Requirements

- macOS (uses `networksetup` and `security` commands)
- Python 3.9+
- [mitmproxy](https://mitmproxy.org/) (`pip install mitmproxy`)

## Installation

```bash
git clone https://github.com/lopezquekk/networkInterceptorCLI.git
cd networkInterceptor
pip install -r requirements.txt
chmod +x ni
```

## Quick Start

```bash
# Start the proxy (requires sudo for CA cert install + system proxy changes)
./ni start

# Browse normally — all traffic is now captured

# See what was captured
./ni list

# Stop the proxy and restore system proxy settings
./ni stop
```

## Usage

### Proxy control

```bash
./ni start         # Start mitmdump on port 8080, install CA cert, set system proxy
./ni stop          # Terminate proxy process, unset system proxy
./ni status        # Show running state, PID, port, and active mock count
```

> `ni start` prompts for your macOS password to install the CA certificate into the System keychain. This only needs to happen once per mitmproxy install.

### Capture inspection

```bash
./ni list                          # List all captured requests (newest first)
./ni list --url api.stripe.com     # Filter by URL substring
./ni get <id>                      # Print full JSON detail of a single capture
./ni clear                         # Delete all captures from the database
```

**Example list output:**
```
a1b2c3d4  2026-06-01T10:23:45+00:00  GET  api.example.com/users  200
e5f6g7h8  2026-06-01T10:23:46+00:00  POST api.example.com/login  200  [MOCKED]
```

**Example `ni get` output:**
```json
{
  "id": "a1b2c3d4",
  "timestamp": "2026-06-01T10:23:45+00:00",
  "method": "GET",
  "host": "api.example.com",
  "path": "/users",
  "status_code": 200,
  "request_headers": "{...}",
  "request_body": "",
  "response_headers": "{...}",
  "response_body": "[{\"id\": 1, \"name\": \"Alice\"}]",
  "is_mocked": 0
}
```

### Mocking endpoints

```bash
# Add a mock: any request matching the glob pattern returns the JSON file
./ni mock add "*/api/users*" /path/to/users.json
./ni mock add "*/api/health*" /tmp/health.json --status 200

# List active mock rules
./ni mock list

# Remove a rule by ID
./ni mock remove <id>
```

Patterns use Unix glob syntax (`*` matches any character sequence). The first matching enabled rule wins.

### iOS Simulator

iOS Simulators on macOS automatically inherit the system proxy, so no extra configuration is needed there. You only need to install the CA cert once per simulator:

```bash
# List booted simulators
./ni simulator list

# Install the CA cert into all booted simulators
./ni simulator install-cert

# Or target a specific simulator by UDID
./ni simulator install-cert --udid 5A74B7E3-C2C0-444A-81B9-4348AC4A399A
```

Boot the simulator first, then run `ni simulator install-cert`. After that, HTTPS traffic from the simulator is fully intercepted and visible in `ni list`.

## File Layout

```
networkInterceptor/
├── ni                  # CLI entry point (executable)
├── addon.py            # mitmproxy addon — intercepts requests, writes captures
├── requirements.txt
└── lib/
    ├── paths.py        # Runtime path constants (~/.networkinterceptor/)
    ├── db.py           # SQLite capture CRUD
    ├── mocks.py        # Mock rules persistence + fnmatch matching
    ├── proxy.py        # Start/stop/status + system proxy + cert install
    └── simulator.py    # iOS Simulator listing + cert installation via xcrun simctl
```

### Runtime data (`~/.networkinterceptor/`)

| File | Purpose |
|------|---------|
| `captures.db` | SQLite database of all intercepted requests |
| `mocks.json` | Persisted mock rules |
| `proxy.pid` | PID of the running mitmdump process |
| `mitmproxy-ca-cert.pem` | CA certificate (auto-generated by mitmproxy on first run) |

## How It Works

1. `ni start` launches `mitmdump` with `addon.py` as a plugin and configures macOS to route all HTTP/HTTPS traffic through `localhost:8080`.
2. On first start, mitmproxy generates a CA certificate. `ni` installs it into the **System keychain** (`sudo security add-trusted-cert`) so all browsers trust it.
3. For each request, `addon.py` checks the active mock rules. If a pattern matches, it returns the local JSON file immediately without hitting the network and marks the capture as `[MOCKED]`.
4. All requests (real and mocked) are written to SQLite for later inspection.
5. `ni stop` sends `SIGTERM` to the mitmdump process and restores the system proxy to its previous state.
